# 2026-06-09 — Watermark bug sweep: Elsevier ScienceDirect + ACM eTOC

## Summary

Systematic audit of journal watermark bugs using consecutive-run DOI overlap analysis on `server_backup_0602`. Identified and fixed 15 journals re-fetching their full RSS window every day due to broken or missing watermarks.

## Root causes found

**ScienceDirect feeds (all `rss.sciencedirect.com`):** No per-entry date fields — only a channel-level `lastBuildDate`. Date-based watermark never advances. Same ~50–100 papers re-fetched every run. Fix: `"id_pattern": "pii/S(\\d+)"`.

**ACM eTOC feeds:** `prism:coverDate` set months ahead (future issue date). No other date available. Watermark stalls at future date. Fix: `"id_pattern": "10\\.1145/(\\d+)"`. (Same as TOSEM, fixed in prior commit.)
