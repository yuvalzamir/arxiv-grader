# Run: 2026-06-03 — Full crash (invalid JSON in Yael's profile)

[[Home]] | [[Pipeline Overview]] | [[AI Pipeline]]

**Status:** No-run — pipeline crashed at startup before any user was processed. Recovered by fixing Yael's profile and re-running.

---

## What happened

The cron job started at 00:30:01 ET. It immediately crashed with a `JSONDecodeError` while iterating all users to determine their fields — before triage, fetching, or any user processing began.

Additionally, `davide-candoli-phys-gmail-com` had already failed the previous day (2026-06-02) due to a missing env var, and was being auto-retried today when the crash occurred.

---

## Root cause

**Broken JSON in `users/Yael/taste_profile.json`** — introduced by a manual profile edit session (2026-06-02) using the `edit-profile` skill.

The `interests_description` field contained unescaped double quotes:
```
non-quantitative ("wet" biology)
```

In JSON, `"wet"` inside a string value terminates the string early. Python's `json.load()` raised:
```
JSONDecodeError: Expecting ',' delimiter: line 6 column 514 (char 622)
```

The crash point in `run_all_users.py`:
```python
user_fields = {u.name: _user_field(u) for u in users}  # line 791

def _user_field(u):
    return json.load(f).get("field", "cond-mat")  # line 175 — crashes here
```

---

## Fix applied

Changed `"wet"` to `'wet'` (single quotes) in Yael's `interests_description`. Deployed via SCP.

**Root fix to skill:** Added a JSON validation step to `edit-profile.md` (Step 7) — after writing the profile, always run:
```bash
python -c "import json; json.load(open('users/<name>/taste_profile.json'))" && echo "Valid JSON"
```

## Lessons

- JSON profiles edited manually (via `edit-profile` skill or otherwise) must be validated before deployment. Unescaped double quotes inside string values are a silent corruption risk.
- Any invalid profile crashes the entire run for all users — the `_user_field()` loop has no per-user error handling.
- Consider adding a try/except per user in `_user_field()` to skip corrupted profiles rather than crashing the whole pipeline.
